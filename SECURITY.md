# Security Policy

## Reporting a Vulnerability

Please do not report security vulnerabilities in public issues.

Instead, use one of these private channels:

- GitHub Security Advisories (preferred), if enabled for this repository

If GitHub Security Advisories is not available, open an issue requesting a
private contact channel and do not include exploit details in public.

Include:

- affected component and platform (Android, iOS, or shared)
- reproduction steps and impact
- logs or screenshots (with secrets redacted)
- suggested fix (optional)

## Response Targets

- Initial acknowledgment: within 72 hours
- Triage decision: within 7 days
- Fix timeline: depends on severity and release risk

## Supported Scope

DebridHub is currently scoped to:

- OAuth2 device flow
- `/rest/1.0/user` account status
- local reminders and diagnostics

Reports outside the current product boundary may be accepted as best effort.

## Disclosure

Please allow time for triage and a fix before public disclosure.
After remediation, coordinated disclosure is welcome.
